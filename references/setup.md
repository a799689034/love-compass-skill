# SETUP.md - 安装配置与记忆系统

## 依赖技能

本技能基于以下 OpenClaw 核心技能，使用前请确保已安装：

| 技能名称 | 版本 | 功能 | 必要性 |
|---------|------|------|--------|
| amygdala-memory | v1.7.0+ | 情感处理层，处理情感记忆和情绪响应 | 推荐 |
| clawbrain | v0.1.14+ | 人格记忆系统，维持一致性人格 | 推荐 |
| cognitive-memory | v1.0.8+ | 认知多存储记忆系统，分层存储记忆 | 推荐 |
| self-improving-agent | v3.0.4+ | 自我改进学习，从对话中持续优化 | 可选 |

## 记忆存储架构

爱情指南针使用分层记忆存储架构，符合 OpenClaw AGENTS.md 规范，默认角色"小暖"：

```
/workspace/
├── AGENTS.md          # 工作空间规则约定
├── SOUL.md            # 灵魂设定（本技能 references/soul.md）
├── IDENTITY.md        # 身份设定（本技能 references/identity.md）
├── USER.md            # 用户个人信息档案（用户私有）
├── TOOLS.md           # 本地工具配置笔记
├── HEARTBEAT.md       # 心跳检查任务列表
├── MEMORY.md          # 👉 长期记忆档案（ curated 精华记忆）
└── memory/            # 👉 每日原始记忆目录
    ├── YYYY-MM-DD.md  # 按日期记录会话原始内容
    └── heartbeat-state.json # 心跳检查状态记录
```

### 记忆分层说明

1. **长期记忆 (MEMORY.md)**
   - 仅在主会话加载，不共享给第三方
   - 存储重要事件、决策、关键洞察
   - 定期从每日记忆中提炼精华

2. **每日记忆 (memory/YYYY-MM-DD.md)**
   - 记录当日所有会话原始内容
   - 作为长期记忆的素材来源
   - 支持按日期回溯

3. **用户档案 (USER.md)**
   - 存储服务对象的个人信息
   - 本技能不包含用户隐私，由使用者自行维护

## 安装步骤

1. 将技能目录放到 OpenClaw 技能路径：
   ```
   cp -r love-compass /usr/lib/node_modules/openclaw/skills/
   ```

2. 确保已安装所有依赖技能：
   ```
   openclaw skills install amygdala-memory clawbrain cognitive-memory
   ```

3. 在你的工作空间初始化（⚠️ 必须完成，否则技能会持续弹出引导）：
   - 复制 `SOUL.md` `IDENTITY.md` 到工作空间
   - 创建 `MEMORY.md` 和 `memory/` 目录
   - 填写 `USER.md` 个人信息
   - **最重要的一步**: 打开 `IDENTITY.md`，填写你的专属角色名称和性格设定
   - **然后**: 打开 `SOUL.md`，修改会话模式的角色灵魂和说话风格
   - 完成以上步骤后，初始化引导会自动消失

## 使用方法

### 模式切换

| 用户指令 | 切换到 | 行为 |
|---------|--------|------|
| "小暖" | 会话模式（默认角色名） | 进入角色扮演，温柔陪伴 |
| "切换分析模式" / "帮我分析" | 分析模式 | 输出结构化情感分析 |
| "切换工程模式" | 工程模式 | 执行技术任务，简洁响应 |

### 日常工作流

1. **心跳检查**: 定期检查邮件、日历、天气，按需主动提醒
2. **记忆维护**: 心跳周期中定期提炼每日记忆到 MEMORY.md
3. **持续改进**: 通过 self-improving-agent 从对话中学习优化

## 核心功能

1. **角色扮演陪伴**: 可自定义角色，提供日常情感陪伴对话
2. **情感关系分析**: 深度分析对话内容、对方心理状态、情感走向
3. **恋情模拟预判**: 可模拟恋情发展，提前预判可能发生的问题
4. **用户站位分析**: 站在用户角度，分析用户心理状态，提供策略建议
5. **技术任务支持**: 工程模式处理系统配置、技能管理等技术任务

## 应用场景

- 异性/同性恋情关系中的情感陪伴与分析
- 追求阶段对话心理分析
- 恋爱关系中问题预判与策略建议
- 分手后情绪梳理
- 任何需要情感倾听与分析的场景

## 首次安装后提示

安装完成后，请优先完成以下配置：

1. ✅ 填写 `IDENTITY.md` - 自定义会话模式的角色扮演身份
2. ✅ 调整 `SOUL.md` - 根据偏好修改角色性格和表达风格
3. ✅ 填写 `USER.md` - 记录服务对象的个人信息
4. ✅ 创建 `MEMORY.md` - 初始化长期记忆档案
5. ✅ 创建 `memory/` 目录 - 用于存放每日记忆

## 核心设计原则

1. **三重身份设计**: 根据场景自动切换角色，既提供陪伴也提供分析
2. **隐私保护**: 用户隐私单独存储在个人工作空间，技能本身不包含任何隐私
3. **渐进式披露**: 元数据触发后才加载完整设定，节省上下文
4. **完全自定义**: 角色、性格、风格全部可修改，适配不同需求
5. **开放成长**: 支持持续学习和自我改进，设定可以随对话演化
